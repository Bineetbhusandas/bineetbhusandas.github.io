04 Featured work — problem → approach → impact

Module Lead · 2025–present

ITT / MMT — Integrated Track & Trace

Next-gen multi-modal (ocean · air · rail · road) visibility platform

Java 21 · Virtual ThreadsSpring Boot 3.5AWS Lambda · SQS FIFO CQRSCaffeineAuth0 · RBAC
Problem

The legacy tracker couldn't scale to multi-modal volumes, every team re-wrote search, security and caching differently, and read traffic was crushing the write database.

Approach & key decisions
  • Read/write datasource routing — separated query load from ingestion so heavy reads never block writes (a CQRS-style split).
  • Universal search framework over 19+ entities — one configurable engine instead of N bespoke search endpoints.
  • Stampede-safe Caffeine caching sustaining ~400 concurrent calls over 300 pooled DB connections.
  • Java 21 virtual threads in a pluggable scheduler framework (7 modules) for massively concurrent I/O without thread-pool tuning.
  • Centralised security, exception, logging & outbound-REST frameworks every module reuses.
Impact
  • Reusable frameworks adopted across the platform — new modules ship in a fraction of the time.
  • Read-heavy workloads served from cache with no write-path contention.
  • Earned a 4.2 / "Exceeds Expectations" appraisal and the Module Lead role.

Backend Engineer → SSE · 2022–2024

Smart Navigator — carrier integration backbone

Ocean container visibility across 50+ carriers

Java · Spring BootAWS LambdaSnowflake MultithreadingRPA framework
Problem

Tracking pulled carriers one-by-one — slow, brittle, and unable to keep up. Coverage gaps meant customers couldn't see their containers, and onboarding each new carrier was bespoke.

Approach & key decisions
  • Re-engineered sequential carrier polling into a parallel, multithreaded pipeline — roughly 10–15× throughput.
  • Built an AWS Lambda eURL service handling ~10–15K requests/day at ~99% success.
  • Wrote 3 carrier adapters from scratch and a reusable RPA framework covering ~16 carriers (~50–100K records/day).
  • Integrated Snowflake (PETA) and a Hapag-Lloyd push API; raised test coverage from ~30% to 85%+.
Impact
  • Order-of-magnitude faster tracking and far wider carrier coverage.
  • Mentored 10+ interns and ~6 new joiners; promoted twice on this platform.
